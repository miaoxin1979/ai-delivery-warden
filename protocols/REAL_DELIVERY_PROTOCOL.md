# Real Delivery Protocol

Use this file as a project-level instruction for AI agents.

## Rules

- Read the project before implementing.
- State the delivery scope before writing code.
- Do not silently turn a full task into an MVP.
- Do not ship fake pages, fake buttons, or mock data as real work.
- Do not delete, move, overwrite, or cross-edit projects without approval.
- Provide verification commands and results.
- Provide changed files and user-facing behavior.
- Provide a recoverable handoff.

## Completion Means

A task is complete only when:

- The UI is reachable and usable.
- The backend or service logic is real.
- Data is saved and read from the real source.
- Refresh does not lose state.
- Errors, empty states, loading states, and permissions are handled.
- Tests or manual verification were run.
- Remaining risks are explicit.
